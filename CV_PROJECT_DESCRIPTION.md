# CV Project Description — SkillForge AI

## Project Name

**SkillForge AI: Career Bootcamp**

## One-line Description

Offline-first AI Engineering career bootcamp app that helps Vietnamese beginners learn AI concepts, complete code missions, build portfolio evidence and prepare for entry-level AI roles.

## CV Summary

Built **SkillForge AI**, a mobile-first AI Engineering career bootcamp app using React, Vite, Tailwind CSS, LocalStorage and Capacitor. The app guides Vietnamese beginners through micro-lessons, code missions, Evidence Vault, Capstone 2.0, career tracks, portfolio export and interview defense workflows.

## Best CV Bullet Points

- Built **SkillForge AI**, an offline-first AI Engineering career bootcamp app for Vietnamese beginners using React, Vite, Tailwind CSS, LocalStorage and Capacitor.
- Designed a job-readiness learning workflow: Learn → Lab → Code Mission → Build Project → Save Evidence → Explain → Export Portfolio.
- Implemented **Evidence Vault**, career tracks, placement test, spaced repetition, weekly sprint review, portfolio export and mock interview defense features.
- Created runnable Python starter kits for AI glossary, CSV insight analysis, ML predictor demo, offline Document Q&A and AI automation workflows.
- Built **Capstone 2.0: Document Q&A Assistant**, simulating a RAG workflow offline with local documents, keyword retrieval, source citation, no-answer behavior and evaluation reporting.

## Interview Pitch

I built SkillForge AI because many beginner AI learning apps stop at short lessons and quizzes. I wanted to create a stricter system that turns learning into evidence. The app guides users through AI concepts, code missions, capstone projects, GitHub proof, CV bullets and interview defense. Technically, it is built as an offline-first React/Vite mobile app with LocalStorage and Capacitor, plus Python starter kits for hands-on practice.

## Target Roles for Portfolio

## Honest Limitations

- MVP does not use a real AI API.
- Capstone simulates RAG offline with keyword retrieval rather than embeddings/vector database.
- No backend, login or cloud sync in the current version.
- Production AI deployment, monitoring and MLOps are represented as learning modules, not fully implemented infrastructure.
